# Executive Summary

## Name
IR Sigma Playbook

## Problem Statement
Provides a structured methodology to respond to alerts triggered by the core Sigma rules in the UIW-CDPv2 environment.

## Objectives
- Rapidly investigate and contain endpoint threats detected by Sigma rules.
- Ensure consistent response steps for execution, credential access, defense evasion, and persistence tactics.

## Compliance
- **NIST CSF 2.0:** DE.AE (Detection Analysis), RS.MI (Response Mitigation)
- **NIST 800-53:** SI-4 (Information System Monitoring), IR-4 (Incident Handling)

## MITRE ATT&CK Framework
- **T1059.001** Command & Scripting Interpreter: PowerShell
- **T1003.001** OS Credential Dumping: LSASS Memory
- **T1070.001** Indicator Removal on Host: Clear Windows Event Logs
- **T1053.005** Scheduled Task/Job: Scheduled Task
- **T1033** System Owner/User Discovery
- **T1218.010** System Binary Proxy Execution: Regsvr32
- **T1105** Ingress Tool Transfer (Bitsadmin)
- **T1047** Windows Management Instrumentation (WMI)
- **T1136.001** Create Account: Local Account
- **T1574** Hijack Execution Flow (RDP Session Hijacking)

## Assumptions and Limitations
- Assumes the analyst has access to Security Onion 3.1 telemetry, SOC console, and EDR/NAC isolation capabilities.

# Analysis

## Monitoring and Notifications
- Alerts are generated by the Detections module in Security Onion.
- Notifications are forwarded via the SOAR Response Agent webhook.

## Playbook Verification
- Validation is achieved through the Phase 5 Adversary-in-a-Box emulation pipeline (`tests/anomaly_simulation/`).

## Recommended Response Action(s)

1. **Preparation & Triage**
   - **Validate the Alert:** Verify if the activity aligns with known IT administrative tasks, scheduled patching, or approved software deployments.
   - **Identify Context:** Identify the affected endpoint(s), the user account associated with the process, and the exact timestamp.
   - **Examine Process Lineage:** Look at the parent and child processes.

2. **Investigation & Analysis**
   - **Payload Decoding:** For PowerShell alerts (Rule 1), extract and decode the Base64 payload to understand the script's intent.
   - **Credential & Session Assessment:** If LSASS dumping or RDP hijacking occurs, assume credentials are compromised. Monitor immediately for lateral movement.
   - **Evaluate Persistence:** Review parameters for newly created Scheduled Tasks or Local Accounts. Identify the schedule frequency and binary execution path.
   - **Network Connections:** If Regsvr32 or Bitsadmin fired, check network logs for remote C2 IPs and payloads.
   - **Log Auditing:** If event logs were cleared, pivot to central ES logs or EDR telemetry.

3. **Containment**
   - **Endpoint Isolation:** Quarantine the compromised host using the SOAR router-block tier (Phase 4) or manual EDR functionality.
   - **Account Suspension:** Disable the compromised AD/local user accounts.
   - **Perimeter Blocking:** Block malicious C2 IPs/domains at firewalls.

4. **Eradication & Remediation**
   - **Terminate Processes:** Kill active malicious RDP sessions, PowerShell tasks, or WMI processes.
   - **Remove Artifacts:** Delete malicious scheduled tasks, registry modifications, and dropped payload files.
   - **Credential Reset:** Force an immediate password reset for all exposed user accounts.
   - **System Scans:** Run deep EDR scans to ensure removal of residual malware.

5. **Recovery & Post-Incident Activity**
   - **Restore & Reconnect:** Reconnect the endpoint to the network only once verified clean. Reimage if deep persistence is suspected.
   - **Rule Tuning:** Update the Sigma rule `falsepositives` or SO Detections exclusions if triggered by legitimate tools.
   - **Root Cause Analysis:** Conduct a post-mortem to determine initial access and close the vulnerability.
