# Commit Approach

## Team Decision: Delegated Commits

After a team vote, the Suburban-SOC team has agreed to use the **Delegated Commits** approach as defined in the CIS 3353 Group Project guidelines (Part 2, Section 6).

## What This Means

- All code contributions are developed by individual team members on their local branches
- Completed work is submitted via pull requests to the `merge-to-main` branch
- The designated **Project Lead (Sterling Garnett)** reviews and merges all pull requests before they reach the main branch
- No direct pushes to `master` are permitted without review

## Rationale

This approach ensures code quality, accountability, and a clean commit history. It mirrors real-world SOC team workflows where changes are peer-reviewed before deployment to production systems.
