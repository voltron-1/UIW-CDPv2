# Plan: Non-blocking wiki sync (replaces agy pre-commit hook)

Date: 2026-07-02 · Owner: Tommy · Branch: feat/so-migration-scaffold

## Requirement
A repo wiki that is constantly updated by default and never blocks any
commit, PR, or merge. Replaces the deleted `agy` pre-commit hook, which
blocked commits with a synchronous LLM call and never actually ran.

## Decision
Deterministic post-merge sync via GitHub Actions. No LLM in the loop (an
AI change-summary layer can be added later behind an `ANTHROPIC_API_KEY`
secret, as a separate non-required job).

## Components
1. `scripts/wiki/build_wiki.py` — stdlib-only generator. Reads
   `docs/migration/` + `docs/adr/ADR-001`, emits wiki pages into an
   output dir:
   - `Home.md` — overview + per-phase progress (done/total from WBS Log lines)
   - `Migration-Status.md` — full task table (ID, objective, issues, status, log)
   - Mirrors: `Migration-Plan`, `Work-Breakdown`, `Salt-Map`,
     `Integration-Inventory`, `ADR-001-Security-Onion-Migration`
     (relative doc links rewritten to wiki page links)
2. `.github/workflows/wiki-sync.yml` — on push to `main` (docs/scripts
   paths) + `workflow_dispatch`. Builds pages, clones
   `<repo>.wiki.git` with `GITHUB_TOKEN`, copies pages in (no deletes of
   unmanaged pages), commits+pushes only on diff. `concurrency` guarded.

## Non-blocking guarantees
- Runs post-merge on `main`; not a required status check; nothing gates on it.
- If the wiki repo is missing/uninitialized, the job prints remediation
  and exits 0.

## Validation
- Local: run generator, inspect output pages.
- Remote: enable `has_wiki`, seed the wiki with the generated pages,
  confirm pages render; after PR #154 merges, confirm the workflow run
  is green and idempotent (second run = "no changes").
